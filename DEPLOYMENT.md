# Deployment Guide - American Pen House

## Vercel Deployment (Recommended)

Vercel is the optimal platform for deploying Next.js applications. It offers:
- Automatic deployments from Git
- Serverless functions for API routes
- Built-in analytics and monitoring
- Free tier with generous limits
- Zero-config deployment

### Step 1: Prepare Your Repository

1. Initialize Git (if not already done):
```bash
git init
git add .
git commit -m "Initial commit: American Pen House website"
```

2. Push to GitHub:
```bash
git remote add origin https://github.com/YOUR_USERNAME/american-pen-house.git
git push -u origin main
```

### Step 2: Deploy to Vercel

1. **Visit [vercel.com](https://vercel.com)**

2. **Click "New Project"**

3. **Import your repository:**
   - Select your GitHub account
   - Search for "american-pen-house"
   - Click "Import"

4. **Configure project:**
   - Framework Preset: Next.js
   - Root Directory: ./
   - Environment Variables: Leave empty (no secrets needed for basic deployment)
   - Click "Deploy"

5. **Wait for deployment** (typically 1-2 minutes)

### Step 3: Set Up Custom Domain (Optional)

1. Go to Project Settings → Domains
2. Enter your domain (e.g., americanpenhouse.com)
3. Follow DNS configuration instructions
4. SSL certificate auto-generated by Vercel

### Step 4: Post-Deployment

1. **Configure Email Notifications:**
   - Update `app/api/contact/route.ts` to use an email service:
     - SendGrid
     - Resend
     - Mailgun
     - AWS SES
   
   Example with Resend:
   ```bash
   npm install resend
   ```

2. **Add Environment Variables to Vercel:**
   - Go to Settings → Environment Variables
   - Add your email service credentials

3. **Update Contact Form Handler:**
   ```typescript
   // In app/api/contact/route.ts
   import { Resend } from 'resend';
   
   const resend = new Resend(process.env.RESEND_API_KEY);
   
   // Send email in POST handler
   await resend.emails.send({
     from: 'noreply@americanpenhouse.com',
     to: 'info@americanpenhouse.com',
     subject: `New Contact Form Submission from ${name}`,
     html: `<p>${message}</p>`,
   });
   ```

## Alternative Deployment Options

### AWS Amplify

1. **Connect repository:**
   - Go to AWS Amplify Console
   - Click "New app" → "Host web app"
   - Select GitHub and authorize
   - Choose repository

2. **Build settings:**
   - Build command: `npm run build`
   - Start command: `npm run dev`

3. **Deploy:**
   - Click "Save and deploy"

### Netlify

1. **Connect repository:**
   - Go to [netlify.com](https://netlify.com)
   - Click "New site from Git"
   - Select GitHub and authorize

2. **Build settings:**
   - Build command: `npm run build`
   - Publish directory: `.next`

3. **Deploy:**
   - Click "Deploy"

### Railway

1. **Install Railway CLI:**
   ```bash
   npm install -g @railway/cli
   ```

2. **Login and deploy:**
   ```bash
   railway login
   railway init
   railway link
   railway up
   ```

## Environment Variables

Create a `.env.local` file for local development:

```env
# Email Service (Example: Resend)
RESEND_API_KEY=your_api_key_here

# Analytics (Optional)
NEXT_PUBLIC_GA_ID=your_ga_id_here
```

**Important:** Never commit `.env.local` to Git. It's already in `.gitignore`.

   />
   <Script
     id="google-analytics"
     strategy="afterInteractive"
     dangerouslySetInnerHTML={{

## Troubleshooting

### Build Fails
1. Check Node version: `node --version`
2. Clear cache: `rm -rf .next node_modules && npm install`
3. Check logs for specific errors

### Deployment Fails
1. Verify environment variables are set
2. Check `.env.local` is not committed
3. Review build logs in Vercel dashboard

### Slow Load Times
1. Check Vercel Analytics
2. Optimize images further
3. Enable edge caching
4. Review bundle size: `npm install -D @next/bundle-analyzer`

### Contact Form Not Working
1. Check API endpoint is deployed
2. Verify environment variables for email service
3. Test with Postman:
   ```bash
   POST http://localhost:3000/api/contact
   Content-Type: application/json
   
   {
     "name": "Test User",
     "email": "test@example.com",
     "message": "Test message"
   }
   ```
